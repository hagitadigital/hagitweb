Experiment · I

Claude Code as OS — קרוסלה מהטרמינל.

איך בניתי קרוסלה של 8 סליידים בלי לעזוב את הטרמינל — כולל 4 תמונות שנוצרו ב-Higgsfield, ובלי לפתוח את VS Code. הכל בפקודות בעברית ל-Claude Code, דרך MCP, עם פלט HTML מוכן ל-export.

Claude Code MCP Higgsfield Nano Banana 2 HTML / CSS
I
The Brief

השאלה שפתחה את הקנבס.

רציתי קרוסלה לאינסטגרם שתסביר ל-followers שלי שמשתמשים ב-AI: Claude Code אינו רק לקוד — הוא קוקפיט שמייצר תוכן, ויזואל, וטקסט. אבל לא רציתי "AI tutorial" — רציתי שהקרוסלה עצמה תהיה התשובה.

Meta-content. תהליך הבנייה הוא הסיפור. פקודה אחת בעברית בטרמינל, ומה שיוצא בצד השני הוא ההוכחה.

II
The Stack

שלושה כלים. שפה אחת.

Claude Code — המוח. רץ בטרמינל בתוך VS Code, כותב את ה-HTML, מנהל את הקבצים, קורא ושומר תמונות. כל פקודה היא לא רק תשובה — היא פעולה.

MCP (Model Context Protocol) — הצינור. הפרוטוקול שמאפשר ל-Claude לדבר עם Higgsfield, Notion, Drive, GitHub, Canva — בלי להעתיק ובלי להדביק. בלי לצאת מההקשר.

Higgsfield — היד שיוצרת. מריצה את Nano Banana 2 (Gemini 2.5 Flash Image של Google) ושולחת 4 תמונות בסגנון אחיד. בלי לצאת מהטרמינל, בלי לפתוח dashboard.

III
The Prompt

הפקודה שייצרה את הכל.

זו הפקודה שכתבתי ל-Claude Code — בעברית. ככה אני באמת עובדת. שיחה, לא syntax. ארבעה דברים חשובים: בריף ברור, אסתטיקה מוגדרת, בחירות טיפוגרפיות, ופלט מובן.

IV
The Generations

ארבע תמונות, שפה אחת.

Claude Code קרא ל-Higgsfield MCP ארבע פעמים. הראשונה נוצרה מאפס — והפכה ל-reference image לשלוש הבאות. ככה שמרתי על אותה זווית, פלטה ו-mood. בלי זה — כל פלט רץ לעולם משלו.

V
The Result

הקרוסלה הסופית.

8 סליידים בקובץ HTML אחד. שניים של פתיחה ומניפסט, ארבע תמונות עם טקסט עברית מוטמע, סלייד של ה-stack, וסלייד סיום שמזמין למעבדה הזו. רנדור ל-PNG דרך headless render, מוכן להעלאה לאינסטגרם.

VI
The Takeaway

מה למדתי שלא מסופר ברשת.

Claude Code הוא לא IDE — הוא orchestrator. ברגע ש-MCP מחבר אותו לכלים אחרים שלך, הוא הופך לקוקפיט שמייצר במקום שמייעץ. ההבדל הזה משנה הכל.

שפה ויזואלית אחידה דרך AI אפשרית — אבל רק אם מעבירים תמונת reference בין רנדרים. בלי זה, כל פלט רץ לעולם משלו, וכל ניסיון לבנות סדרה נכשל.

עברית בתמונות עדיין נשברת ב-Nano Banana 2 ברוב הזמן. הפתרון: ליצור תמונות נקיות, להוסיף את הטקסט ב-HTML / CapCut / Figma אחרי. שילוב human × AI על הטיפוגרפיה. זה לא פגם — זו הכרה בגבולות.

How MCP Connects to Claude
Behind the Magic

MCP — הצינור שמחבר את הכל.

Model Context Protocol הוא הסטנדרט שמאפשר ל-Claude לדבר עם כלים חיצוניים בלי שתצטרכי לעבור מסך, להעתיק תוצרים, או לנהל API keys ידנית. אלה השלושה דברים שצריך להבין — ואלה ה-servers שמותקנים אצלי בסטודיו.

01

מה זה MCP server?

חתיכת קוד קטנה שיודעת לדבר עם כלי חיצוני (Higgsfield, Notion, Drive, GitHub, וכו') ולתרגם בקשות מ-Claude לפעולות בכלי. הוא רץ ברקע במחשב שלך, או בענן.

02

איך מחברים אותו ל-Claude Code?

קובץ הגדרות claude_desktop_config.json (או .mcp.json בפרויקט). מוסיפים שורות שמצביעות ל-server הרצוי. Restart לטרמינל — וזהו, Claude יודע להפעיל אותו.

03

איך קוראים ל-MCP מתוך שיחה?

לא צריך syntax מיוחד. את כותבת בעברית: "תיצרי תמונה ב-Higgsfield" — Claude מזהה את הכוונה ומפעיל את ה-server הרלוונטי. ברקע. הכל הופך לפעולה.

04

מה החוזקה האמיתית?

הקשר מצטבר. Claude זוכר מה יצא מה-server אחד, מעביר את זה הלאה לשני, ובונה זרימת עבודה שלמה. במקום 5 כלים מנותקים — סטודיו אחד.
